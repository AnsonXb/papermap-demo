    价值与意义
    三个维度的价值主张
    💡

    个人效率价值

    将文献综述时间从 2~3 周压缩到 3~5 天。节省研究生至少 40+ 小时的重复劳动。通过智能推荐阅读路径，先读 3 篇奠基论文，再按需扩展，避免盲目阅读。

    🌱

    社会公益价值

    降低科研新人的入门门槛，让更多学生能快速进入研究状态。帮助识别低质量的重要论文在关键节点位置，促进学术公平。中文友好设计，服务国内 300 万+ 在读研究生群体。

    💰

    商业发展潜力

    目标用户群体明确且有付费意愿。可拓展到企业技术调研场景（调研团队需要快速了解某个技术领域）。数据积累后可形成"学术知识图谱数据库"，成为科础基础设施。

    MVP 开发计划
    三周交付可用产品，聚焦核心功能快速验证
    Week 1 — 数据层 + 后端
    API 对接 数据库
    调通 Semantic Scholar API 获取论文列表及引用关系 → 搭建 Neo4j 图数据库定义节点（论文）和边（引用）的数据结构 → 编写数据清洗脚本爬取论文并清洗存入图数据库 → 接入 TRAE API 批量总结论文（控制成本：只总结 Top 50）→ 实现 PageRank 算法计算每篇论文的权重分数 → 提供 /api/search、/api/graph、/api/summary 三个接口
    Week 2 — 前端 + 可视化
    前端开发 可视化
    搭建 Next.js 项目完成初始化和配置 → 实现搜索页面（输入框 + 加载动画）→ 集成 Cytoscape.js 实现引用关系图谱展示（节点大小=重要性、颜色=论文类型、可点击查看详情）→ 实现节点交互功能（点击节点显示论文详情 + AI 总结内容）→ 阅读路径可视化（高亮推荐的阅读顺序，带箭头动画）→ 响应式适配确保移动端也能用
    Week 3 — AI 增强 + 打磨
    AI 能力 打磨优化
    AI 根据图谱结构生成 Markdown 格式的综述大纲（按时间线或技术路线组织论文）→ 智能阅读清单（根据用户背景——本科/研究生——推荐难度梯度）→ 键导出综述大纲（Markdown 格式）→ 分享功能：生成精美的图谱截图 + 分享卡片 → Demo 视频录制：演示"3 分钟看懂 Transformer 发展史"→ 部署上线（Vercel 前端 + Railway 后端）→ 准备答辩 PPT
    目标用户
    精准定位三大核心用户群体

    核心用户：正在写毕业论文/开题报告的研究生

    最刚需的用户群。导师要求"先看 100 篇论文把领域摸清楚"，但不知从何下手。PaperMap 让他们在 3 天内完成原本需要 2 周的文献调研工作。

    次要用户：刚转入新研究领域需要快速建立知识体系的科研工作者

    跨领域研究者、需要技术调研的工程师。需要快速了解某个研究方向的全貌，找到关键论文和前沿进展。

    潜在用户：撰写综述论文的学者 / 需要技术调研的工程师生

    看到一篇新论文想知道它的学术背景和后续影响，需要快速定位与自己工作最相关的论文。
